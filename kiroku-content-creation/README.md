# Kiroku Content Creation

This directory is the working space for all written content produced for the Kiroku website — blog posts, case studies, and supporting editorial material.

## What this is

Kiroku is a project at the intersection of archival practice and machine learning. The content produced here tells the story of that work: where it started, what was built, what it got right, and what remains genuinely uncertain.

This is not a marketing library. It's a record of a real project — and the writing should reflect that.

## Workflow overview

1. **Ideation** — identify the story worth telling and why it matters now
2. **Research** — gather notes, verify timeline, record sources
3. **Draft** — write a rough first pass, prioritising narrative over polish
4. **Rewrite** — read aloud, cut what doesn't earn its place, sharpen the argument
5. **Edit** — line-level pass for clarity and consistency with the style guide
6. **Fact-check** — flag every specific claim (names, dates, organisations, quotations) and confirm before publishing
7. **Publish** — only after sign-off on facts

## A note on fact-checking

Several details in the project history — conference names, exact dates, funding announcements — still need to be confirmed against primary sources. Placeholders in the form `[TO CONFIRM]` mark anything unverified. **Do not publish content with unresolved placeholders.**

If you are using AI assistance to write or edit content in this repository, read `AGENT_BRIEF.md` first.
